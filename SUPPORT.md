# Support

## Getting Help

Use the following channels depending on the kind of help you need:

- Bug reports: open a GitHub issue using the bug report template.
- Feature ideas: open a GitHub issue using the feature request template.
- Questions about local setup or contribution workflow: start with README.md, LOCAL_DEV_README.md, and CONTRIBUTING.md.

## Before Opening an Issue

Please check:

- README.md for project overview and setup.
- LOCAL_DEV_README.md for local development, Docker Compose, and debugging details.
- CONTRIBUTING.md for coding, branch, and pull request expectations.
- Existing issues to avoid duplicates.

## What to Include

When asking for help, include:

- Your operating system.
- The command you ran.
- The relevant log output or error message.
- The component involved, such as frontend, backend, Docker, or docs.
- Steps to reproduce the problem.

## Security Issues

Do not use public issues for vulnerabilities or exposed credentials. Follow SECURITY.md instead.

## Maintenance Scope

The main maintained surfaces are the backend, frontend, local development workflow, and contributor tooling. The android and twa directories are retained for reference only and are not under active development.
